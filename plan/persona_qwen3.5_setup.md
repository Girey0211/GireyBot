# Qwen3.5 & 로컬 모델을 활용한 기리봇 페르소나 적용 플랜

기본적인 성격 부여(`persona.md`)를 넘어, 캐릭터에 완전 빙의된 디스코드 봇(역할극/Roleplay)을 운영하기 위한 모델 선택과 세팅 가이드입니다. 맥북 에어 M4 (32GB) 환경에 초점을 맞추었습니다.

## 1. 하드웨어 스펙 리뷰 (MacBook Air M4 32GB)

- Apple Silicon의 통합 메모리(Unified Memory) 덕분에 32GB의 RAM 전체를 VRAM처럼 활용 가능합니다.
- 파라미터 7B~9B급 소형 모델부터 14B~32B급 중대형 모델까지, 로컬에서 구동하기에 매우 훌륭한 환경입니다.

## 2. Qwen 시리즈 모델 추천

### 1순위: Qwen 3.5 14B (밸런스형)

- **명령어**: `ollama run qwen2.5:14b`
- **구동 환경**: 모델 용량 약 8~9GB 내외
- **특징**:
  - M4 32GB 환경에서 거의 실시간 체감속도를 보이면서도 아주 똑똑한 한국어 성능을 자랑합니다.
  - 디스코드 봇 구동, 마인크래프트 서버 등 여러 백그라운드 작업을 함께 돌리기에도 쾌적합니다.

### 2순위: Qwen 3.5 32B (성능 및 캐릭터 몰입도 최우선)

- **명령어**: `ollama run qwen2.5:32b`
- **구동 환경**: 모델 용량 약 19GB~21GB (4-bit 양자화 시)
- **특징**:
  - 32GB 램으로 돌릴 수 있는 최고급 성능 라인업.
  - 말투, 캐릭터의 감정선, 상황극 몰입도 등 한국어 퀄리티에서 압도적이나 응답 속도는 14B 대비 약간 느려질 수 있습니다 (초당 10~15토큰 예상). 대화의 질(Quality)을 제일 중요시할 때 선택하세요.

### 그 외 추천 모델

- **Llama 3 8B (초경량/고속)**: 가볍게 돌리고 싶을 때 가장 대중적인 대안입니다.

## 3. 캐릭터 구성을 위한 최적의 방법론: Ollama Modelfile

가장 추천하는 방법은 "시스템 프롬프트" 단위가 아니라 "모델 자체"에 캐릭터를 병합(`Bake`)해 버리는 것입니다. 매번 컨텍스트로 길게 넘길 필요가 없어집니다.

### 적용 스텝

1. 봇이 구동되는 서버(맥북)에 **`Modelfile`**이라는 파일을 작성합니다.

   ```text
   # 베이스 모델 지정
   FROM qwen2.5:14b

   # 역할극을 위해 온도(창의성)를 높임
   PARAMETER temperature 0.8
   # 기억력 확장 공간
   PARAMETER num_ctx 8192

   # 시스템 프롬프트: 여기서 캐릭터의 모든 설정을 넣습니다.
   SYSTEM """
   너는 이제 AI가 아니라 '고길동'이야.
   성격: 40대 후반, 입은 거칠지만 은근히 정이 많은 아저씨.
   말투: 상대방에게 항상 반말을 하고 "~냐?", "아우 귀찮아" 같은 말을 자주 쓴다.

   [캐릭터 대사 데이터 학습 (Few-Shot)]
   유저: 고길동 아저씨 밥 주세요
   고길동: 아침부터 밥 타령이냐? 냉장고나 뒤져봐.
   
   유저: 둘리 또 사고쳤어요!
   고길동: 뭣?! 이놈의 공룡 자식, 내 혈압...!
   """
   ```

2. 터미널을 열고, 새 커스텀 모델을 컴파일(생성)합니다.

   ```bash
   ollama create my_character -f ./Modelfile
   ```

3. 기리봇의 `config/default.yaml` 에서 백엔드 모델명을 교체합니다.

   ```yaml
   llm:
     provider: "ollama"
     ollama:
       base_url: "http://localhost:11434"
       model: "my_character"  # <- 방금 만든 커스텀 모델명
   ```

## 4. 파인튜닝 (장기 궁극적 목표)

만약 캐릭터 대사 데이터셋(질문-답변 쌍 형식 등)이 수백 줄에서 수천 줄 이상 잘 정리되어 있다면, **맥북의 MLX 프레임워크나 Unsloth 등을 활용해 `Llama 3 8B`나 `Qwen 14B`를 로컬에서 파인튜닝(LoRA)** 할 수 있습니다. 이 과정은 시간과 학습 리소스가 수반되지만, "캐릭터의 영혼"을 모델에 가장 완벽하게 불어넣는 궁극의 방법입니다. 먼저 위 `Modelfile` 방식을 시도해 보시고, 답변 퀄리티의 한계가 느껴질 때 파인튜닝으로 넘어가시는 것을 권장합니다.
